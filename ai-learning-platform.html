Project Overview

What I built and why.

SoloLeveling is a full-stack AI-powered DSA (Data Structures & Algorithms) learning platform I built to solve a real problem: existing platforms like LeetCode use static, pre-written problems. I wanted something that generates fresh content on demand — different every time — using GPT-4.

The platform covers 6 core DSA chapters, each with 10 progressive difficulty levels. At every level, GPT-4 generates a unique coding problem and a detailed concept explanation in the learner's chosen language. Users submit their solution and get AI feedback on correctness, efficiency, and style.

The backend is a Flask REST API with a content cache (to avoid redundant API calls) and a fallback system when OpenAI is unavailable. It's deployed on Render with a Python-based CI/CD pipeline.

🤖

GPT-4 problem generation

Every problem is generated fresh by GPT-4 — no static question bank. Each prompt is tailored to the chapter topic, difficulty level, and chosen programming language.

📈

10-level progressive difficulty

Each of the 6 chapters has 10 difficulty levels from beginner to expert, letting learners build genuine mastery before advancing.

✅

Live code validation

Submit your solution and receive AI feedback: correctness check, time/space complexity analysis, and style suggestions — all generated by GPT-4.

⚡

Smart content caching

Concepts and questions are cached in-memory after first generation, cutting API costs and response time for repeat visits to the same chapter/level.

Technical Architecture

How it works under the hood.

Backend (Python Flask · Render)

  • GET /api/chapters — all 6 DSA chapters
  • GET /api/chapters/<id>/concept — GPT-4 explanation
  • GET /api/chapters/<id>/questions/<level> — AI problem
  • POST /api/chapters/<id>/validate — code review
  • In-memory ContentCache + OpenAI fallback system
  • Deployed on Render via render.yaml

Frontend (HTML / JS / CSS)

  • Multi-page app: dashboard, learning, roadmap, jobs, profile, upload
  • JavaScript ES6+ — Fetch API for async calls to Flask backend
  • Language selector: Python, Java, JavaScript, C++, C#
  • Dynamic problem rendering — no page reloads
  • Progress tracking per chapter across sessions

What I Learned

Why this project stands out.

Building this taught me how to design a real AI-powered REST API at production level — caching strategies to control OpenAI costs, fallback systems for API downtime, and structuring prompts that return consistent, parseable output from GPT-4.

The frontend/backend split also deepened my understanding of async JavaScript, CORS configuration, and deploying Python services to Render with environment-variable management. This is the project that made me comfortable owning an entire full-stack AI product from scratch.
